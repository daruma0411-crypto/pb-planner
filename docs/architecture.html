PB企画プランナー

アズワン向け PB（プライベートブランド）企画支援チャットボット
仕入れ先の製品データをもとに、AIと壁打ちしながらPB企画を進めるツール

Flask + Python Claude Sonnet 4 Railway Function Calling

0 まず知っておきたい用語

このドキュメントで使われるAI・技術用語を、身近なたとえで解説します。

LLM（大規模言語モデル）

Large Language Model

大量のテキストデータから言語パターンを学習したAI。人間のように文章を理解し、生成できます。

たとえ：世界中の本・論文・Webページを読んで育った、超博識な相談相手。ただし自分の記憶は持たない。

Claude Sonnet 4

Anthropic社の LLM モデル

本システムで使用しているLLMの具体的な製品名。Anthropic社が開発。複数のモデルサイズ（Haiku＜Sonnet＜Opus）があり、Sonnetはコストと性能のバランス型。

たとえ：ChatGPTの「GPT-4」にあたるもの。車でいえば車種名（カローラ、プリウス等）のようなもの。

システムプロンプト

System Prompt

AIの「役割」「ルール」「性格」を定義する、ユーザーには見えない裏側の指示文。本システムでは約400行の詳細な指示でAIを「PB企画コンサルタント」に仕立てています。

たとえ：新人社員への研修マニュアル。「あなたはPB企画のプロです。お客様にはこう対応してください」という業務指示書。

Function Calling（ツール呼び出し）

Tool Use

AIが「データベース検索」「Excel生成」などの外部機能を自分で判断して呼び出す仕組み。AIは文章生成だけでなく、システムの機能を操作できるようになります。

たとえ：秘書が「調べておきますね」と言って自分で資料室に行って調べてくるイメージ。指示しなくても文脈から必要な行動を判断する。

API（エーピーアイ）

Application Programming Interface

ソフトウェア同士が通信するための「窓口」。本システムはClaude APIを通じてAIにテキストを送り、回答を受け取ります。

たとえ：レストランの注文カウンター。メニュー（仕様）に従って注文（リクエスト）すると料理（回答）が出てくる。

セッション

Session

ユーザーがシステムを使い始めてから終わるまでの「一連のやり取り」を記憶する仕組み。会話履歴やPBカードの状態を保持します。

たとえ：カフェでの打ち合わせ。席を立つまで（1時間）ノートに書いた内容が残る。別の日に来たら白紙からスタート。

JSONL（製品データ形式）

JSON Lines

1行に1製品の情報が入ったテキストファイル。Excelの1行＝1製品と同じ考え方で、プログラムが高速に読み書きできます。

たとえ：1行1枚の製品カード。Excelシートの行がそのままテキストファイルになったもの。

Flask（フラスク）

Python Webフレームワーク

Pythonでwebアプリを作るための軽量フレームワーク。URLごとに処理を割り振り、ブラウザからのリクエストに応答します。

たとえ：店舗の受付係。来店したお客様（ブラウザ）を適切な窓口（機能）に案内する。

システムプロンプトの核心ルール

約400行の指示文で、AIを「PB企画のプロ」に仕立てています。

ルール内容普通のチャットAIとの違い
聞き返すな、動け 文脈から分かれば確認せず即座にツール実行 普通のAI: 「FLS-1000を検索しますか？」
本システム: 即検索して結果を表示
前の会話を忘れるな 製品名・型番・価格・分析結果を全て記憶 普通のAI: 前の話題を忘れがち
本システム: 全ての情報を引き継ぐ
一度に処理しろ 複数の値が指示されたらまとめて処理 普通のAI: 1つずつ確認
本システム: 3件同時にPBカード更新
ユーザーの書式に従え 「連番ふって」なら通し番号を振る 普通のAI: 自分の形式で出力
本システム: ユーザー指示の形式に従う

4 ユーザーワークフロー

ユーザーがPB企画を完成させるまでの典型的な流れです。

製品検索・選定

「トミー精工のオートクレーブを見せて」→ AIが製品DBを検索し、候補を提示。ベース製品を決定。

PBカード作成

アズワン品番・販売価格・JANコード・メーカー型番・入数・キャッチコピーの7項目をAIと対話しながら埋めていく。

フレームワーク分析

3C分析・SWOT・ポジショニングマップ・5Forces・価格帯マップをワンクリックで実行。チャートも自動描画。

仕様壁打ち

「8番のチャンバー径をΦ460に変更」→ AIが仕様更新＋変更ログを自動記録。全変更履歴が追跡可能。

アウトプット生成

PIM Excel / 企画書Word / 英訳Excel / カタログHTML の4種をボタン一つで自動生成・ダウンロード。

8 セキュリティ・制限事項

安全に運用するための設計上の考慮事項です。

🔒 セキュリティ対策

  • APIキーは環境変数で管理（コードに埋め込まない）
  • セッションは1時間で自動削除
  • 古いセッションファイルは100件超で自動クリーンアップ
  • HTTPS通信（Railway標準）

⚠️ 既知の制限

  • 同時利用 — ワーカー1のため、同時に1ユーザーのみ処理
  • セッション — 1時間で期限切れ（長時間作業は注意）
  • AI応答時間 — 複雑な分析は最大120秒かかる場合あり
  • データ規模 — 大量製品の全文検索には非対応（現状数百件規模）
